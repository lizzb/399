[Tentative] Spec / Feature set

On this page, I will document the planned/actual functionality of my I-Day app, as well as specifying non-goals and providing justification / factors influencing these decisions if necesssary.

Goals / Guiding Principles

My goal in this independent study is to develop a web/mobile hybrid app that can both ease and enhance the experience of the Career Fair, specifically the SWE iDay career fair that happens at Northwestern every October.

Reference to help me when designing and iterating on the UI and UX of this app:
  • Simplify the career fair experience by enabling the user to do everything a paper program could do - and more - through the app.
  • Increase engagement before, during, and after the career fair through this app.
  • Focus on the experience of the student first.
  • Mobile = first, tablet = second, others = afterthought (priorities of smooth user interface experience)

Technology decisions (some pending)

In Scope

Features that are planned to be implemented and fully[ish] functional.
  • Sort: Ability to sort (ascending and descending) company list by:
    • Name, alphabetical
    • Location - room/floor, alphabetical; table/booth, numeric
    • Company size/number of employees, numeric
  • Filter: Ability to filter company list by:
    • [Live] Search - start with company names only (--> might also provide sort/filter by room and floor abilities)
    • Targeted majors
    • Targeted job type/role (Intern, Coop, Full-Time, Masters)
    • Type of company / Industry categorization
    • Locations hiring - group by state or region, TBA
    Programmatically generate all filters to maximize reusability and minimize chance of typos/bugs
  • Saved user settings/preferences, so that the same search critera do not have to be repeatedly entered.
  • Ability locate a company's booth location on a map. That is, looking at a company's profile should allow you to jump to a map with a marker. This would ideally also work in reverse, selecting a table on the map could bring up the option of seeing that company profile.
  • Ability to star favorite companies
  • Ability to hide companies that are not applicable to the user from cluttering up / displaying on the map

Out of Scope

Due to the limitations of this being an independent study, as well as time constraints, the following features will intentionally be not considered in order to focus on delivering a more manageable and higher-quality set of in-scope features.
  • Designing the user experience / user interface for non-students, i.e.:
    • Recruiters / Company representatives
    • SWE student coordinators / planners of IDay
    • Potential high-level stakeholders, e.g. McCormick, University Career Services, etc.
While out of scope for functionality implementation, it is possible that I will be able to prototype the experience of some of the below features. As the project evolves, this may change.
  • Ability to easily import new data into the application for non-techy people (i.e. event planners) - this allows the app to be reused with little additional effort
  • Minimize internet usage
